## 3. 模块详细设计

### 3.1 状态机设计

```
                    ┌──────────┐
                    │  初始状态  │
                    └─────┬────┘
                          │ 扫描设备
                    ┌─────▼────┐
                    │  设备发现  │
                    └─────┬────┘
                          │ 选择设备
                    ┌─────▼────┐
                    │  配对确认  │
                    └─────┬────┘
                          │ 配对成功
                    ┌─────▼────┐
         ┌─────────►│  已连接   │◄──────────┐
         │          └─────┬────┘           │
         │                │ 录音           │
         │          ┌─────▼────┐           │
         │          │  录音中   │           │
         │          └─────┬────┘           │
         │                │ 停止           │
         │          ┌─────▼────┐           │
         │          │  待处理   │           │
         │          └─────┬────┘           │
         │                │ AI处理         │
         │          ┌─────▼────┐           │
         │          │  处理中   │           │
         │          └─────┬────┘           │
         │                │ 完成           │
         │          ┌─────▼────┐           │
         │          │  有结果   │───────────┘
         │          └──────────┘
         │  断开连接
    ┌────▼────┐
    │  已断开  │
    └─────────┘
```

### 3.2 数据流设计

```
用户操作 → 事件触发 → 状态更新 → UI 渲染
    │          │          │          │
    ▼          ▼          ▼          ▼
  按钮点击  → 录音开始  → 状态变化  → 波形绘制
  设备选择  → 配对流程  → 连接成功  → 启用模块
  停止录音  → AI处理   → 处理进度  → 结果展示
```

### 3.3 模拟数据策略

| 数据类型 | 生成方式 | 说明 |
|----------|----------|------|
| 设备列表 | 预定义 + 延迟渲染 | 模拟逐个发现设备 |
| 信号强度 | 基础值 + 随机波动 | 模拟真实信号变化 |
| 电量 | 基础值 + 缓慢下降 | 模拟耗电 |
| 录音波形 | Web Audio API 实时采集 | 真实音频数据 |
| AI 处理结果 | 预定义场景模板 | 根据场景选择 |
| 性能数据 | 真实计时 + 随机变化 | 记录每次操作 |

## 4. 交互设计

### 4.1 关键交互流程

#### 4.1.1 设备连接流程
```
打开页面 → 点击"扫描设备" → 扫描动画 → 设备列表出现
→ 点击"连接" → 配对码弹窗 → 确认配对 → 连接成功动画
→ 启用所有模块 → 跳转到监控页面
```

#### 4.1.2 录音到结果流程
```
切换到录音页 → 点击麦克风 → 请求麦克风权限
→ 波形实时显示 → 录音计时 → 点击停止
→ 自动跳转AI处理页 → 点击"开始处理"
→ 处理进度动画 → 完成 → 跳转结果页
```

### 4.2 动画规范

| 动画 | 类型 | 时长 | 缓动 |
|------|------|------|------|
| 页面切换 | fadeIn | 0.4s | ease |
| 设备发现 | slideIn | 0.4s | ease |
| 扫描脉冲 | scanPulse | 2s | infinite |
| 按钮悬停 | transform | 0.3s | ease |
| 录音脉冲 | pulse | 1s | infinite |
| 进度条填充 | width transition | 0.5s | ease |

## 5. 性能优化策略

### 5.1 渲染优化
- 使用 `requestAnimationFrame` 进行波形绘制
- Canvas 尺寸使用 2x 分辨率适配 Retina 屏幕
- 非可见区域暂停动画

### 5.2 音频优化
- AudioContext 复用，避免频繁创建
- AnalyserNode 使用合适的 FFT 大小（2048）
- 及时释放不用的音频资源

### 5.3 内存优化
- 录音结束后关闭 AudioContext
- 清除 setInterval 和 requestAnimationFrame
- 避免内存泄漏
